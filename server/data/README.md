# Система коэффициентов

## Как использовать

### 1. Добавление ваших данных

Замените содержимое файла `coefficients.json` на ваш полный JSON файл с 13000 строк.

### 2. Структура данных

JSON должен иметь следующую структуру:

```json
{
  "products": {
    "system_key": {
      "categories": {
        "category_name": {
          "widths": [0.1, 0.2, 0.3, ...],
          "heights": [0.1, 0.2, 0.3, ...],
          "values": [
            [val1, val2, val3, ...],
            [val4, val5, val6, ...],
            ...
          ]
        }
      }
    }
  }
}
```

**Где:**

- `system_key` - уникальный ключ системы (например, "uni1_zebra", "uni2_roller", и т.д.)
- `category_name` - категория ткани (например, "E", "1", "2", "3", "4", "5")
- `widths` - массив значений ширины в метрах (по возрастанию)
- `heights` - массив значений высоты в метрах (по возрастанию)
- `values` - двумерный массив коэффициентов [высота][ширина]

### 3. Настройка систем в БД

В таблице `systems` добавьте поле `system_key`, которое должно соответствовать ключам в JSON файле.

Например:

- Имя системы: "Uni1 Zebra Classic"
- System Key: "uni1_zebra"

### 4. API Endpoints

#### Получить коэффициент

```
POST /api/coefficients/calculate
{
  "systemKey": "uni1_zebra",
  "category": "E",
  "width": 1.5,
  "height": 2.0
}
```

**Успешный ответ (точное совпадение):**

```json
{
  "coefficient": 14.2684,
  "systemKey": "uni1_zebra",
  "category": "E",
  "usedSystemKey": "uni1_zebra",
  "usedCategory": "E",
  "isFallbackCategory": false,
  "width": 1.5,
  "height": 2.0,
  "warning": null
}
```

**Успешный ответ (использован fallback категории):**

```json
{
  "coefficient": 14.2684,
  "systemKey": "uni1_zebra",
  "category": "XYZ",
  "usedSystemKey": "uni1_zebra",
  "usedCategory": "1",
  "isFallbackCategory": true,
  "width": 1.5,
  "height": 2.0,
  "warning": "Категория \"XYZ\" не найдена, использована \"1\""
}
```

**Ошибка (система не найдена):**

```json
{
  "message": "Система \"unknown_system\" не найдена в файле коэффициентов",
  "systemKey": "unknown_system",
  "category": "E",
  "usedSystemKey": null,
  "usedCategory": null
}
```

#### Получить список систем

```
GET /api/coefficients/systems
```

**Ответ:**

```json
{
  "systems": ["uni1_zebra", "uni2_roller", ...]
}
```

#### Получить категории для системы

```
GET /api/coefficients/systems/:systemKey/categories
```

**Ответ:**

```json
{
  "systemKey": "uni1_zebra",
  "categories": ["E", "1", "2", "3", "4", "5"]
}
```

#### Получить диапазоны размеров

```
GET /api/coefficients/ranges?systemKey=uni1_zebra&category=E
```

**Ответ:**

```json
{
  "systemKey": "uni1_zebra",
  "category": "E",
  "widthRange": { "min": 0.1, "max": 3.0 },
  "heightRange": { "min": 0.1, "max": 3.0 }
}
```

## Как работает интерполяция

Система использует **билинейную интерполяцию** для нахождения коэффициентов между узлами сетки.

Если запрашиваемые ширина/высота не совпадают с точками в массиве, коэффициент рассчитывается автоматически на основе ближайших 4 точек.

### Пример:

Если в данных есть:

- Ширины: [1.0, 1.5, 2.0]
- Высоты: [1.5, 2.0, 2.5]

А вы запрашиваете width=1.3, height=1.8, система найдет коэффициент между точками (1.0,1.5), (1.5,1.5), (1.0,2.0), (1.5,2.0) используя интерполяцию.

## Использование в формулах

Пример использования в формуле заказа:

```typescript
const coefficient = await getCoefficient(
  system.systemKey,
  fabric.category,
  order.width / 100, // перевод см в метры
  order.height / 100
);

const price = coefficient * multiplier + additionalCosts;
```

## Производительность

- JSON файл загружается один раз при первом запросе
- Данные кэшируются в памяти
- Поиск и интерполяция выполняются быстро (< 1мс)
- Подходит для работы с большими объемами данных (13000+ строк)

## 🔄 Автоматический поиск ближайшего коэффициента (Fallback)

Система автоматически ищет подходящий коэффициент по следующему алгоритму:

### 1. Поиск системы (systemKey)

1. **Точное совпадение** - ищем точный ключ системы
2. **Игнорирование регистра и пробелов** - если не найдено, ищем с нормализацией строки
3. **Ошибка** - если система не найдена вообще, показывается ошибка

### 2. Поиск категории ткани

1. **Точное совпадение** - ищем точную категорию (например, "E")
2. **Игнорирование регистра** - ищем без учета регистра ("e" → "E")
3. **Игнорирование пробелов** - убираем пробелы и ищем снова
4. **Fallback на первую категорию** - если категория не найдена, автоматически используется первая доступная категория для этой системы

### 3. Поиск по размерам

- Для размеров используется **билинейная интерполяция**
- Если запрошенные размеры не совпадают с узлами сетки, коэффициент рассчитывается автоматически на основе ближайших 4 точек

### Примеры работы fallback:

**Пример 1: Категория с другим регистром**

- Запрошено: `systemKey: "uni1_zebra", category: "e"`
- Найдено: `category: "E"`
- Результат: ✅ Коэффициент рассчитан

**Пример 2: Категория с пробелами**

- Запрошено: `systemKey: "uni1_zebra", category: " 1 "`
- Найдено: `category: "1"`
- Результат: ✅ Коэффициент рассчитан

**Пример 3: Несуществующая категория**

- Запрошено: `systemKey: "uni1_zebra", category: "XYZ"`
- Найдено: `category: "1"` (первая доступная)
- Результат: ⚠️ Коэффициент рассчитан с предупреждением
- Пользователь видит: "Использована ближайшая категория"

**Пример 4: Несуществующая система**

- Запрошено: `systemKey: "unknown_system", category: "E"`
- Результат: ❌ Ошибка, система не найдена

## ⚠️ Устранение проблем

### Ошибка "Система не найдена"

**Проблема:** При создании заказа система не может найти ключ системы в файле коэффициентов.

**Причины:**

1. **Неправильный `system_key`** - ключ системы в базе данных не соответствует ключу в `coefficients.json`
2. **Отсутствующие данные** - система вообще не представлена в файле `coefficients.json`

**Решение:**

### 1. Проверка доступных систем

```bash
# Получить список всех system_key из coefficients.json
GET /api/coefficients/available-keys

# Ответ:
{
  "systemKeys": ["uni1_zebra", "uni2_roller", "uni3_classic", ...]
}
```

### 2. Проверка категорий для системы

```bash
# Получить список категорий для конкретной системы
GET /api/coefficients/available-categories?systemKey=uni1_zebra

# Ответ:
{
  "systemKey": "uni1_zebra",
  "categories": ["E", "1", "2", "3", "4", "5"],
  "count": 6
}
```

### 3. Правильная настройка системы

В разделе "Справочники → Системы":

1. **Имя системы** - произвольное название (например, "Uni1 Zebra Classic")
2. **System Key** - **должен точно соответствовать** ключу из `coefficients.json` (например, "uni1_zebra")
3. **Множитель** - коэффициент для расчета итоговой цены

❗ **Важно:** `system_key` чувствителен к регистру и должен полностью совпадать!

### 4. Правильная настройка тканей

В разделе "Справочники → Ткани":

1. **Название ткани** - произвольное название (например, "Blackout Premium")
2. **Категория** - **должна существовать** для выбранной системы в `coefficients.json` (например, "E", "1", "2", и т.д.)
3. **Тип ткани** - roll, zebra и т.д.

❗ **Важно:** Категория должна быть **точно** такой же, как в файле коэффициентов!

### 5. Диагностика проблемы

#### Шаг 1: Проверить systemKey системы

```sql
-- Проверить, какой systemKey установлен для системы
SELECT id, name, system_key FROM systems WHERE id = 'your_system_id';
```

#### Шаг 2: Проверить, существует ли systemKey в файле

```bash
GET /api/coefficients/available-keys
```

Убедитесь, что `system_key` из базы присутствует в списке.

#### Шаг 3: Проверить категорию ткани

```sql
-- Проверить категорию ткани
SELECT id, name, category FROM fabrics WHERE id = 'your_fabric_id';
```

#### Шаг 4: Проверить, существует ли категория для системы

```bash
GET /api/coefficients/available-categories?systemKey=your_system_key
```

Убедитесь, что категория ткани присутствует в списке.

### 6. Тестирование коэффициента

Попробуйте вручную рассчитать коэффициент:

```bash

{
  "systemKey": "uni1_zebra",
  "category": "E",
  "width": 1.5,
  "height": 2.0

```

**Успешный ответ (200):**

```json

  "coefficient": 14.2684,
  "systemKey": "uni1_zebra",

```

**Ошибка (404):**

```json
{
  "message": "Коэффициент не найден для системы \"uni1_zebra\", категории \"E\"",
  "systemKey": "uni1_zebra",
  "category": "E"

```

## 🔧 Автоматическая защита

Система автоматически предупреждает о проблемах с коэффициентами:

1. **При вводе данных створки** - если коэффициент не найден, появляется уведомление
2. **Визуальная индикация** - створка с отсутствующим коэффициентом подсвечивается красным
3. **Блокировка создания заказа** - нельзя создать заказ со створками без коэффициента
4. **Детальное сообщение об ошибке** - указывается, для какой системы и категории коэффициент отсутствует
